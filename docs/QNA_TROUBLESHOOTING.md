# 이슈 및 Q&A 해결 가이드

## 1. 이슈 해결: `articles` 데이터가 비어있는 현상

### 원인 파악
현재 Supabase 데이터베이스에 접속해보면 실제로는 10개의 데이터가 저장되어 있지만, `npm run dev` 시 브라우저나 터미널에는 `[]` (빈 배열)이 반환되는 현상이 발견되었습니다.
이는 **Row Level Security (RLS)** 설정 때문입니다. `src/app/page.tsx`에서 데이터를 불러올 때 읽기 전용 접근 키인 `anon` 키(클라이언트)를 사용했는데, 현재 Supabase의 `articles` 테이블에 공개 읽기(SELECT)를 허용하는 RLS 정책이 없기 때문에 데이터가 반환되지 않고 조용히 빈 배열을 반환한 것입니다.

### 해결 계획 및 실행
이 프로젝트는 대시보드이며 현재 Next.js 16 App Router의 **Server Component**에서 동작하고 있습니다.
따라서 서버 환경에서만 실행되는 로직에서는 RLS를 우회할 수 있는 `service_role` 키를 안전하게 사용할 수 있습니다.
`page.tsx`의 클라이언트 호출을 `supabase`에서 `getServiceSupabase()`로 변경하여 서버 측에서 권한 제약 없이 모든 데이터를 올바르게 Fetch 하도록 코드를 수정했습니다.

## 2. 질문에 대한 답변

### 질문 1: React Query를 쓸 일이 있을까?
**답변: 현재 구성에서는 필수는 아니지만, UX(사용자 경험) 향상 목적이 크다면 추후 도입할 수 있습니다.**
- **현재 상황:** Next.js 16의 App Router에서는 기본적으로 Server Component 내부에서 `await fetch()` 또는 Supabase 클라이언트를 사용해 데이터를 직접 불러오는 것이 표준입니다 (현재 이렇게 구성되어 있습니다).
- **언제 써야 할까?** 
  1. 클라이언트 측에서 **"무한 스크롤(Pagination)"**이나 검색/필터링을 부드럽게 처리하고 싶을 때.
  2. 사용자가 카드를 삭제하거나 북마크할 때, 페이지 전체를 새로고침(`revalidatePath`)하지 않고도 **"즉각적인 낙관적 UI 업데이트(Optimistic UI)"**를 보여주고 싶을 때.
  3. 백그라운드 데이터 갱신(Polling) 기능이 필요할 때.

### 질문 2: n8n 트리거 동작 시 UI가 실시간 업데이트 되는지?
**답변: 아닙니다. 현재 구조에서는 실시간으로 자동 갱신되지 않습니다.**
- **원인:** n8n이 Supabase 데이터베이스에 몰래(Background) 데이터를 삽입하더라도, 브라우저의 화면(Next.js Server Component 렌더링 결과)은 이미 다운로드된 정적 상태이므로 변경 사실을 알지 못합니다. 새로고침을 해야만 최신 데이터를 가져옵니다.
- **해결/개선 방안:**
  1. **실시간 구현 (Supabase Realtime):** 클라이언트 컴포넌트에 Supabase Realtime 채널(`postgres_changes`)을 연결해두면, DB에 새 데이터가 INSERT 될 때마다 브라우저가 이벤트를 감지하여 자동으로 화면에 새 카드를 띄울 수 있습니다.
  2. **수동 갱신 (현재 방식 유지):** 지금 구현된 우측 상단의 "안읽음 동기화" 버튼을 클릭할 때, 작업 완료 후 Next.js의 `revalidatePath('/')`를 호출시켜 화면을 갱신합니다.
